Git y GitHub

Las 4 sentencias más utilizadas

Git init (Inicializar)

Este comando se utiliza para iniciar un nuevo repositorio en un directorio específico. Es como decirle a Git, "¡Vamos a empezar a rastrear cambios en estos archivos!" Puedes verificar si se ha creado el repositorio al observar una carpeta oculta llamada .git en ese directorio. Git guarda en esta carpeta toda la información sobre tu proyecto y los cambios que haces.

Git status (Estado)

Utilizamos este comando para obtener una lista de los archivos en tu proyecto y su estado actual. Si has hecho cambios en tus archivos, git status te mostrará cuáles archivos se han modificado o agregado desde el último commit. Es como una lista de verificación para saber qué está pasando en tu proyecto.

Git add (Agregar)

Este comando no te mostrará nada en la pantalla, pero es crucial. Lo que hace es poner los archivos en un estado llamado "staging" o escenario. Básicamente, estás diciéndole a Git que estos archivos están listos para ser parte del próximo commit. Puedes usar git add de diferentes maneras, como para añadir todos los archivos con una extensión específica o todos los archivos en un directorio.

  • git add *.jpg agregará todos los archivos cuyo nombre termine en .jpg al área de stage.
  • git add ruta/a/directorio agregará todos los archivos del directorio al área de stage.
  • git add . agregará todos los archivos modificados y/o no rastreados al área de stage.

Git commit (Confirmar)

Una vez que has puesto los archivos en el escenario con git add, este comando toma esos archivos y los guarda oficialmente en tu historial de cambios. Es como tomar una "foto" de tus archivos en ese momento. Necesitas agregar un mensaje descriptivo con -m para que sepas más adelante qué cambios hiciste en ese commit. Esto es importante para entender la historia de tu proyecto en el futuro.
